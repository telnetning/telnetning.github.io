在个人博客或者公司中，常常有共享文件的需求。windows 的共享场景有限，因此考虑使用 HTTP 服务器进行共享。
这里使用 Apache 服务器进行共享。这个很容易实现，即在 Apache 的配置中，将目录配置成可浏览即可。

Options +Indexes

但是这个共享的页面，默认格式很难看，尤其是可能文件名无法显示完全，影响使用。所以考虑定制一下这个页面。
其实最简单的想法，就是找到这个页面的模板，然后自己更改 css 就行了。
额，考虑到好几年没碰过 css 了……
然后找一个模板就行了啊，于是找到了abba项目
具体效果图可见仓库源码中的 example 目录。
很简单的一个项目，提供了很多模板，而且可以一行代码搞定……如果你是在国外的话。
这个项目提供了一个脚本，脚本会从网上上下载一些需要的 css、html、Js 文件等。但是在某些网无法下载。于是想自己从项目的 github 上下源码，然而发现源码需要经过预处理器才能形成我们需要的文件。这个预处理就是通过 github pages 生成的——所以只能通过 http 访问页面才能获取到。
没办法，只能手动通过浏览器去把需要的所有文件扒下来。
具体操作如下：

  1. 下载 install
  2. 在 rootDocuments 目录下执行 sh install
  3. 进入 .abba ,将里面的 shtml、css、js 手动替换。下载路径在 install 脚本最后的命令中有，一一对应，其中 css 文件只需要用一个你想要的主题替换就行了。
  4. 将 css 文件中的第一个分号前的从 googleapi 获取字体的代码删除掉，否则影响页面加载

cacheZH

为了减轻上述工作量，我将所有的最终需要的文件下载下来了，然后 fork 了该仓库，在自己的仓库中新建了cacheZH目录，然后添加了最终需要的文件。
然后通过 gh-pages 分支作为 project pages 的原理，托管了这些文件。
总之，现在同样可以一键安装整个项目了：

#进入rootDocuments
T=$(mktemp) && curl -sL https://telnetning.github.com/abba/install -o $T && sh $T

其它的诸如配置 httpd.conf 文件等不再详述。
