vagrant 管理虚拟环境

今日偷得半日闲，学习了一下 vagrant。vagrant 是一个构件虚拟环境的工具。什么意思，就是用来定义一个虚拟机的。但是它本身不具备虚拟化的能力，它的底层需要 virtualBox、Vmware 或者 AWS 之类的工具的支持。你可以把它当作一个脚本，这个脚本规定了虚拟化中的虚拟机是什么样的、能干什么。

能用来干什么呢？很简单，定义一个虚拟环境。这个虚拟环境可以随意移动，可以放到各种系统各种虚拟化环境中来跑。就是你可以拿着一个虚拟机当文件一样拷贝、分发、共享。
这样的好处是：

  • 团队很容易有一个相同的开发环境，彻底杜绝“在我的机器上跑的好好的”这种问题，因为我把环境都一同拷贝给你了。
  • 对于个人，很容易构件一个你需要的开发环境，你可以在一个机器上跑若干个环境。
  • 对于运维人员来说，当然也是很好的，你不再需要一个个机器上去配环境。

了解 Docker 的人一看就知道了，这货跟 Docker 差不多嘛。事实基本是这样的，但是两者之间还是有一些不一样的。总的来说，Docker 更加轻量级一些，但是这两者比起传统虚拟机来说都很轻量级。两者的适用范围也是有差别的。通过 google vagrant vs docker 能找到很多资料。

我们首先安装好 vagrant，同时我们也安装好 virtualBox。之所以用 virtualBox 时因为它是免费的，当然你也可以使用 Vmware 等。

使用 vagrant command -h 还能特定的命令的用法。

vagrant 也是根据已有的镜像来进行构建。vagrant 中把别人打包好的镜像叫做 box。通过 vagrant box 命令我们可以对 box 做各种操作。

首先我们下载一个 box，本来我们可以直接用 vagrant init 加上 url 来直接从网上下载一个 box，但是由于国内的网络环境，我们事先下好 box。在这个 地址 可以找到各种 box，选择你想要的，然后自己想办法 download 下来，这个过程，没有 vps 或者没有好的人品，就在吃饭睡觉前做吧。

下好后，我们可以开始了。

首先添加 box，比如我们下载的文件放在 ~/vagrant/box 中，叫做 archlinux-x64-2014-01-07.box，那么我们这样添加：

vagrant box add arch_base ~/vagrant/box/archlinux-x64-2014-01-07.box

其中 arch_base 是我们给该 box 起的名字。
然后我们使用 vagrant box list 能够看到我们已经添加的 box。
添加之后，我们要准备启动一个 machine（虚拟机实例），首先我们选定一个目录，将其作为当前 vagrant 的工作目录。比如我建立了一个目录为 ~/vagrant/project/arch_01/，我们进入 arch_01 目录，然后

vagrant init

就是在该目录下生成了一个 Vagrantfile，也就是将该目录作为了一个 vagrant project 的目录。

这里我们说说 Vagrantfile，它是虚拟环境的配置文件，它决定了虚拟环境时什么样的。

在 Vagrantfile 中有一行：

config.vm.box = "base"

它决定了我们虚拟环境使用哪个 box，我们将其改成我们要使用的 box，这里改成我们前面添加的 arch_base。当然我们也可以在初始化目录的时候直接指定该 box，就像这样：

vagrant init arch_base

然后我们就可以启动虚拟机了。

vagrant up

启动之后我们可以看看我们的虚拟机，

vagrant global-status

大概得到下面的结果

id       name      provider   state    directory
-----------------------------------------------------------------------------------
68ed028  default   virtualbox poweroff /Users/telnetning/vagrant/project/arch_01

这里的id就是虚拟机的唯一 id，虚拟机的默认 name 就是 default，但是虚拟机的 name 是可能重复的。我们对虚拟机的所有操作都可以根据其 id 来操作，在那么不重复的时候，也可以使用 name。
然后我们就可以 ssh 进虚拟机了，

vagrant ssh 68e

这样就进去上面我们启动的虚拟机了。我们就拥有了一个 linux 虚拟机。

当然，vagrant 提供的功能远不止这么简单。比如我们说说下面几个功能：

一个是设置虚拟机的端口转发，我们在虚拟机的配置文件中可以配置宿主机到虚拟机的端口转发。
需要做的只是在 Vagrantfile 的配置中加入这样一句：

    config.vm.network :forwarded_port, guest: 80, host: 8080

这样就将对宿主机的 8080 端口的访问转发到虚拟机的 80 端口中去了。

再一个就是文件共享功能，这个功能默认已经是打开的。在 linux 虚拟机中，根目录下有一个 /vagrant 的目录，它跟宿主机的某个目录是对应共享的，宿主机和虚拟机都能对这个目录进行操作。这个目录默认就是当前的 vagrant 目录，当然我们也可以在 Vagrantfile 中修改。

另外，在一个 Vagrantfile 中我们也可以定义几个虚拟机，比如我这儿这样定义了两个虚拟机：

15   config.vm.provision :shell, inline: "echo A"
16   config.vm.define "ad_hoc" do |ad_hoc|
17     config.vm.provision :shell, inline: "echo B"
18     ad_hoc.vm.box="arch_base"
19   end
20
21   config.vm.define "algorithm" do |algorithm|
22     config.vm.provision :shell, inline: "echo C"
23     algorithm.vm.box="arch_base"
24   end

这样就定义了两个虚拟机，分别叫做 ad_hoc 以及 algorithm，然后我们使用 vagrant up ad_hoc 这样的命令就可以打开虚拟机。

而对于我们修改过的文件，我们可以使用 vagrant package 打包后分发给别人。并且 vagrant 还提供 share 功能，可以直接将 box 共享到网上去，供全世界的人使用。

使用 vagrant 的好处是很多的。举个例子，我之前在 mac 上构建 hexo 的写作环境不容易，现在我有了 vagrant，我立马弄了一个虚拟环境用来写博客，就是上面的那个 blog 虚拟机。

我来讲讲我的步骤，就当复习下操作：

首先我建立了 arch_blog 当作我的工作目录。在这个目录下，我这样

vagrant init arch_base

使用 arch_base 当作原始镜像。然后我启动了虚拟机：

vagrant up blog

接着 ssh 进去：

vagrant ssh blog

接下来我就有了 archlinux 的环境，按照我之前多端编辑那篇文中的步骤建立好 hexo 的目录。然后就可以启动 hexo 了。
有一个问题，我 hexo 监听的端口是 localhost:4000，为了在宿主机上能够访问，我需要进行端口映射。这样：

config.vm.network :forwarded_port, guest: 4000, host: 4000

就是将宿主机的 4000 端口请求转发到该虚拟机的 4000 端口上，然后我就可以在宿主机傻姑娘使用 localhost:4000 访问虚拟机的 4000 端口了。

然后为了编辑文件方便，当我在虚拟机中建立一个新的文章文件之后，我会将其 mv 到共享目录，也就是 /vagrant 目录中，然后我在宿主机，也就是我的 mac 中打开。什么好处呢？英文博客文章都是 markdown 文件，这样我就可以使用 mac 上的 Mou 神器来进行 markdown 文件的编辑了。编辑完成后直接保存，然后在虚拟机中把文件从 /vagrant 目录移回到 hexo 的博客文章目录中去，就 ok 了。

更大的好处是，有了这个虚拟环境，我以后要是又有了新的机器，我不用在新的机器上再次安装 hexo 的环境（实在太痛苦这个过程）。我只需要安装 vagrant，然后把当前机器中的该 blog 虚拟环境打包，然后在新的机器上使用 vagrant 打开即可。
