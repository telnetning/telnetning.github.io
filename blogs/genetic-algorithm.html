遗传算法，英语为 Genetic algorithm，简称 GA 算法，属于进化算法中的一种。

遗传算法一般应用于人工智能领域，是用于解决最优化问题的一种搜索启发式算法。

简单点理解遗传算法：

  • 目的是为了解决最优化问题，比如求复杂函数的极值
  • 求解方式就是在可行空间搜索
  • 搜索并不是遍历搜索，而是模仿生物进化过程来优化搜索

这就是遗传算法的本质，明显算法的重点和难点就在于如何模仿生物的进化过程来进行启发式搜索。我们把生物的进化过程进行如下描述：

这里有一个初始种群，然后具有某种性状。初始种群经过杂交、变异形成新一代种群，但是新一代种群必须经过自然选择才能得到可以再次进行生殖的成熟种群，所以，这里自然选择放在了新一代种群前面，而不是新一代种群与杂交之间。

一个例子是，很久很久以前，在山的那边海的那边有一群长颈鹿，噢，准确的说，或许该叫短颈鹿。它们起先都是吃草的，所有一切看起来很美好。后来草没了，它们必须去更高的地方找食物--树叶。这个时候它们中间脖子较为长的就活了下来，而其它的都饿死了。这群脖子比较长的生出一些小短颈鹿，这就是杂交，它使得脖子比较长这个性状保留了下来。随着低处的叶子也慢慢被吃，短颈鹿的脖子需要越来越长。这样一代代自然选择，杂交，剩下的短颈鹿都变成了长颈鹿。当然，这其中还参合着我们没有提起的变异的过程。

这里有几个概念：种群，性状，自然选择，杂交以及变异。我们将其试着映射到数学空间中去：

  • 种群，其实就是随机的初始可行解，解空间中的随意一些解
  • 性状，用来对种群中的个体进行选择，在求极值的问题中，就是极值，可以用来评估一组解的好坏，从而对解进行选择
  • 自然选择，用来对解进行选择，选择更好的解
  • 杂交，就是两组解之间进行变换 s
  • 变异，一组解内进行小范围的变化

将生物的进化过程用于遗传算法，大致的流程图如下：

这也就是遗传算法的基本过程，初始化，然后进行 Evalation（评估选择），Crossover（杂交），mutation（变异），不同的是，生物进化没有终点，而在算法中，我们需要设定迭代次数。当迭代超过设定次数时，跳出迭代。

具体遗传算法的实现过程：

  • 首先考虑将建立性状到基因的映射，所谓基因，也就是变量，最常用的做法是将变量映射到二进制空间
  • 交叉（杂交），就是将多个二进制比特串分成几段，然后随机组合
  • 变异，就是将比特串中的某个比特进行反位，若是 \(1\) 则变成 \(0\)，若是 \(0\) 则变成 \(1\)
  • 选择，将性状表现（函数值）比较好的挑选出来

这里，以一个例子说明遗传算法：

\begin{eqnarray} max \quad &f( x_1,x_2 ) = 21.5 + x_1sin( 4 x_1 ) + x_2sin( 20 x_2 ) \\ s.t. \quad &-3.0 \leq x_1 \leq 12.1 \\ &4.1 \leq x_2 \leq 5.8 \end{eqnarray}

编码

编码就是将 \(x_1\) 和 \(x_2\) 映射到二进制空间。这里首先计算 \(x_1\) 和 \(x_2\) 的可行解范围：\(x_1\) 的范围大小为 \(12.1 -（-3）=15.1\)，为了使用二进制表示，并且保证二进制的位数合适，这里将范围大小乘以 \(10000\)，即 \(x_1\) 的取值范围为 \(0\) 到 \(151000\)，同理 \(x_2\) 的解范围为 \(17000\)。

由于 \(2^{17} < 151000 < 2^{18}\)，所以用 \(18\) 位的bit串来表示 \(x_1\)，这样 \(x_1\) 的取值范围为 \(000000000000000000\) 到 \(111111111111111111\)，即 0 到 \(2^{18}-1\)，这样根据比值计算从二进制到实际值的计算：


$$x = {-b \pm \sqrt{b^2-4ac} \over 2a}.$$

这样就将 \(x_1\) 映射到二进制空间了，同理可以将 \(x_2\) 映射到二进制空间。

为了将 \(x_1\) 和 \(x_2\) 在计算时候耦合起来，必须将 \(x_1\) 和 \(x_2\) 的值关联到一起进行搜索运算，也就是将 \(x_1\) 和 \(x_2\) 放到同一个染色体上。

而对于每一条染色体（\(x_1\), \(x_2\)），需要计算性状 \(f(x_1, x_2)\) 的时候，就需要将 \(x_1\) 和 \(x_2\) 从二进制转化回实际的值，这里是实现该过程的 python 代码如下：

def bin_str_dec(s):
    #get the real var value from binary string
    #from chromosom's genotype to phenotype
    tail_str = [0]*15
    for i in range(0,15):
        tail_str[i] = s[i+18]

    x1 = -3 + cal_str_dec(s,18)*15.1/(pow(2,18)-1) 
    x2 = 4.1 + cal_str_dec(tail_str,15)*1.7/(pow(2,15)-1) 
    return (round(x1, 6),round(x2, 6))

适应度计算

输入：种群

输出：新的相同数目的种群

适应度计算就是对每一条染色体计算其性状，也就是对每个二进制题特串计算其函数值，然后根据值的大小进行选择。

需要注意的是，在进行选择之前，我们需要先初始化一个种群，也就是若干组解。这里初始化十个 \(33\) 位的比特串，作为初始解空间。

def Init(len):

然后，计算出各个比特串的对应的函数值，函数值越大的说明性状越好，越应该被保留，反之函数值越小的越应该被淘汰。而遗传算法的思想就是概率选择，如何保证函数值越大的拥有越大的概率呢？这里使用了一种“赌轮盘法则”，本质就是将各个比特串的概率映射到 \([0, 1]\) 上的一块空间，函数值大的比特串占的块就大。想象为 \([0, 1]\) 一块蛋糕：

如图，不同的染色体（比特串）拥有不同的份额。具体要完成这个转换，需要一下几步：

  • 将各个比特串的函数值都加起来，记为 \(F=_{k=1}^{10}f(v_k)\)
  • 计算各个函数值与 \(F\) 的比值，得到 \(p_{k}\)，这就是各个块了: \(p_k=\)
  • 为了划出块并且便于选择，把 \(q_k\) 记作前 \(k\) 个 \(p_k\) 的和,即为 \(q_k=_{j=1}^{k}p_k\)

完成之后，要朝这个盘子上扔石头，扔到谁就选中哪一个比特串，这是通过随机数来实现，产生 \((0, 1)\) 的随机数，若产生的数小于 \(q_1\)，则说明选中了第一块，若产生的数介于 \(q_{k-1}\) 与 \(q_k\) 之间，则说明选中了第 K 块。

将 \(p_k\) 和 \(q_k\) 换到坐标上或许更容易理解，下面是一个五个比特串的情况：

这里输入的 s 就是十个比特串的 list（对应其它语言的数组），需要注意的是，这里的 biggest_order 是最优的个体（二进制串），再每一轮的迭代中，不管是适应度选择计算，交叉还是变异，都必须保持最有的个体不受变化，这样才能保证下一轮的最优解不小于这一轮。所以上述代码第三行的 if 语句就是直接 copy 最优的个体到下一轮而不做选择。在交叉和变异过程中也有同样的处理。

交叉

输入：种群

输出：相同个体数量的种群

交叉就是对个体之间进行基因重组。再交叉过程中，有一个交叉概率的参数。假设现在有两个个体的染色体，要对它们进行交叉运算，首先先根据交叉概率判断是否进行交叉：此处我们将交叉概率 \(p_c\) 设置成 \(0.25\)，也就是有 \(\) 的概率实际进行交叉。这样对于个体数目为是 10 的种群，可以写出下面的伪代码：

打乱该数组（用于取个体下标随机配对）   
for i from 0 to 4 
    if array[i] != 上轮最优个体下标 and array[9-i] != 上轮最优个体下标：

上轮最优个体也就是适应度选择过程中传回的 biggest_order，这里使用如下代码实现：

def crossover(s, last_bigst_order):
    pc = 0.25
    order_list = [0] * 10
    for i in range(0,10):
        order_list[i] = i
    random.shuffle(order_list)
    for i in range(0,5):
        if (order_list[i] != last_bigst_order) and (order_list[(9-i)] != last_bigst_order):
            if random.random() < pc:   
                (s[order_list[i]],s[order_list[9-i]])=cross_two_chrom(s[order_list[i]],s[order_list[9-i]])
    return s

这里是使用随机序列来进行随机配对，具体就是产生一个 \(10\) 个元素的数组，随机存储 \(1\) 到 \(10\)，然后对称选择头尾的元素作为要选中配对的个体的下标。

第八行的 if 语句用来避免最优的个体被交叉。

这里第九行产生 \([0, 1]\) 的随机数，若小于 \(p_c\) 就说明概率被选中，两个个体的基因进行交叉运算，否则尽管配对，但是不交叉。

第十行是调用了交叉运算的函数。交叉运算的方式也有很多中，从个体来看，可以多个个体进行交叉运算，这里只用简单的两个个体。从交叉段数来看，可以将染色体分成多段交叉，这里也只使用两段交叉法。一个简单的图例如下：

这里我们采用的是将 \(33\) 比特的二进制串分成 17bits+16bits，然后两条染色体交叉生成两个新的个体。具体代码实现如下：

def cross_two_chrom(s1,s2):
    #crossover two chromosome
    #exchange the first 17 bits
    s1_tmp = "" 
    s2_tmp = ""
    for i in range(0,17):
        s1_tmp += s2[i]
        s2_tmp += s1[i]
    for i in range(17,33):
        s1_tmp += s1[i]
        s2_tmp += s2[i]
    return (s1_tmp,s2_tmp)

需要注意的是，python 中的字符串不能被改变，所以需要将生成的比特串赋值给新生成的字符串，然后返回。

变异

输入：种群

输出：新的相同个体数目的种群

变异和交叉不同的是，变异是针对个体染色体（二进制比特串）上的每一个基因（位）进行处理，处理方式为简单的反位，也就是原 \(0\) 变 \(1\)，原 \(1\) 变 \(0\)。

变异也有一个参数，记作 \(p_m\)，若概率命中，则该位置反，否则不变。一般变异概率比较小，这和自然的进化过程中的变异是一致的。

在解决具体数学问题上，变异其实是在小范围内搜索的过程，而交叉其实是在大范围内转移然后搜索的过程。

开始跑起来

主程序，只需要依次调用初始化，适应度计算，交叉，变异过程，然后给出循环次数即可，经过多次实验，我的代码大概在 \(3000\) 次内就都能得到最优解。
主函数如下：
