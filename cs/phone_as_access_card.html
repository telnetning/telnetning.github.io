手机用作门禁卡

关于如何用手机当作门禁卡来进出主楼。
前两天在 freebuf 上看到了几篇文章：

  • RFID Hacking：看我如何突破门禁潜入 FreeBuf 大本营
  • 如何利用 Nexus 5 伪造一张门禁卡

这个想一想并不难，门禁卡应该没那么严格，读完这几篇文章大概知道怎么回事了：带有 NFC 功能的手机就可以当作一张 NFC 卡，使用这张卡复制合法的门禁卡的部分数据，就可以充当门禁卡使用。

  • 门禁卡的读卡器一般只读取卡的 ID 值，所以更改手机 NFC 信息的 ID 号就可以了。
  • 手机 NFC 卡的信息可以通过修改配置文件修改。
  • 修改手机配置文件需要 root 权限，也就是需要先 root 手机。

基本的步骤在上面列出的两篇文章中都有，我这里只是贴一下我的整个过程。我的手机是锤子的坚果手机，如果你是别的手机，应该更容易一些，在 root 和修改配置文件的时候不会有那么多事。

0x00

最先的事情当然是先找一张合法的卡，读取卡的信息。在手机上下载 NFC 软件来读取卡信息，我这里下载了两款软件，分别是 NFC Tools 以及 TagInfo，实际上一款应该就够了，但是为了获得更多的信息，我一般两个都用一遍。在这里将其称为 NFC APP。
打开 NFC APP，把合法卡贴到手机后面，然后就可以看到卡的一些信息：

其中有一行 Serial number，就是卡的 ID 号，在其它 NFC 读取软件上也许叫 ID，总之就是卡的标识。如图，上面的卡的标识号就是:

72:d3:c3:1e

当然这里为了安全，我使用的是一张废卡号来写文章，这个 ID 不能进主楼的。：）

卡的标识号是写在 IC 卡的 0 扇区的起始地址的，所以如果想要用卡复制一张门禁卡，也挺简单的。用一张空白卡，简称白卡，将其 0 扇区的标识号改写成合法的卡的标识号就 ok 了。读写卡的读卡器淘宝上有卖。但是需要注意的是，除了需要注意 IC 卡的种类外，还需要注意并不是所有的白卡都是能写的。理论上，所有的卡制造商都应该保证卡标识号的唯一性，并且不允许 0 扇区的标识号被写。但是，中国商家，加上淘宝，无所不能，所以当你去国外网站查询相关资料时候，可能经常能看到 Chinese manufacturer 的字眼，哈哈。

0x01

接下来需要对手机进行操作了，这个时候为了读取到你的手机的 NFC 信息，你需要另外一个支持 NFC 的手机，也装上上面的 MFC APP，推荐 TagInfo，NFC Tools 在这里有可能显示信息不完整，甚至无法读出标识号。这个时候，其实就是把自己的手机当作 NFC 卡，用另一个手机来读取其标识号。

打开 NFC APP，然后将两个手机向上面的图那样背靠背的贴在一起。这里可能需要自己多尝试下，两个手机才能有反应。我使用的是坚果和锤子 T1，刚开始很难相互识别到，经过摸索，终于知道了，坚果的 NFC 芯片在手机中段，T1 的 NFC 芯片在手机上段，用坚果的中段贴合 T1 的上段会有很好的识别状况。
然后就如图第一张图一样，这个时候就读取到了自己手机的标识号。我的读出来是 01, 02, 03, 04。
接下来 root 手机，这一过程不表，不同的手机不同，锤子 root 很不容易的，最后借用乐第三方的 KingRoot 才勉强 root 了，但是貌似也没完全 root。

修改手机配置文件

首先进入手机，把和 nfc 相关的文件 copy 出来，windows 上可以使用 sftp 工具，我这里直接 Linux 下 ssh 进去，然后用 scp 把 /etc 下和 nfc 相关的文件 copy 出来分析。

ls -al /etc | grep nfc

然后把搜索到的文件 scp 出来，大概有这么几个文件：

-rw-r--r--  1 telnetning telnetning 14699 Oct 17 21:44 libnfc-brcm.conf
-rw-r--r--  1 telnetning telnetning 26590 Oct 17 21:51 libnfc-nxp.conf
-rw-r--r--  1 telnetning telnetning  3102 Oct 17 21:44 nfcee_access.xml
-rw-r--r--  1 telnetning telnetning  8989 Oct 17 21:44 nfc-nci.conf

然后直接在里面搜索手机的 NFC 的标识号，

grep -rn "01, 02, 03, 04" *

得到结果：

libnfc-nxp.conf:435:        33, 04, 01, 02, 03, 04,

然后打开文件看看:

# Core configuration settings  
# It includes 
# 18        - Poll Mode NFC-F:   PF_BIT_RATE 
# 21        - Poll Mode ISO-DEP: PI_BIT_RATE 
# 28        - Poll Mode NFC-DEP: PN_NFC_DEP_SPEED 
# 30        - Lis. Mode NFC-A:   LA_BIT_FRAME_SDD 
# 31        - Lis. Mode NFC-A:   LA_PLATFORM_CONFIG 
# 33        - Lis. Mode NFC-A:   LA_SEL_INFO 
# 50        - Lis. Mode NFC-F:   LF_PROTOCOL_TYPE 
# 54        - Lis. Mode NFC-F:   LF_CON_BITR_F 
# 5B        - Lis. Mode ISO-DEP: LI_BIT_RATE 
# 60        - Lis. Mode NFC-DEP: LN_WT 
# 80        - Other Param.:      RF_FIELD_INFO 
# 81        - Other Param.:      RF_NFCEE_ACTION 
# 82        - Other Param.:      NFCDEP_OP 
NXP_CORE_CONF={ 20, 02, 2B, 0D, 
        18, 01, 01,  
        21, 01, 00, 
        28, 01, 00, 
        30, 01, 08, 
        31, 01, 03, 
        33, 04, 01, 02, 03, 04, 
        50, 01, 02, 
        54, 01, 06, 
        5B, 01, 00, 
        60, 01, 0E, 
        80, 01, 01, 
        81, 01, 01, 
        82, 01, 0E 
        }

这里33表示信息类型，LA_SEL_INFO 中的 SEL 应该就是序列号的意思，04 代表的是数据长度，这里明显为 4，然后我们将其中的 01, 02, 03, 04, 改成我们需要的标识号就行了，也就是合法卡的标识号，72, d3, c3, 1e, 然后保存退出。
接下来将更改过的这个文件 push 回手机。但是由于 Android 的 /etc 目录是链接到 /System/etc 的，而 /System 目录是只读的，所以需要重新挂在 /System 为可读写：

adb root
adb remout
adb push libnfc-nxp.conf /etc/

但是我锤的手机难以 root，导致我的 root 功能可能并不完全。首先就是没法 adb root，安装了 adbd Insecure 之后解决了。然后又不允许 remount，最终通过以下命令解决了。

adb shell mount -o rw,remount /System

顺利 adb push 回文件后，重启手机，检验 NFC 信息已经更改，刷门测试 ok。
